# Phase 1: 分析

## 目的

收集项目上下文，调研相关技术和竞品，头脑风暴解决方案，推荐最优方案。

## 涉及角色

- **产品经理** — 需求分析、用户场景梳理
- **架构师** — 技术可行性评估
- **AI 写作专家** — AI 功能相关调研（如适用）

## 步骤

### 1. 上下文收集

- 阅读 `CLAUDE.md` 和 `AGENTS.md` 了解项目规范
- 阅读相关现有代码，理解当前实现
- 明确需求的具体范围和约束

### 2. 调研

- 调研相关技术方案
- 如涉及 AI 功能，参考竞品（Sudowrite、NovelAI）的做法
- 评估技术可行性和工作量

### 3. 头脑风暴

- 列出至少 2-3 个可行方案
- 分析每个方案的优缺点
- 考虑对现有系统的影响

### 4. 推荐方案

- 选择最优方案并说明理由
- 标注关键风险和依赖
- 估算涉及的文件和模块

## 产出物

- 需求分析文档（简洁的 markdown 格式）
- 推荐方案说明
- 风险和依赖清单

## 完成标准

- [ ] 需求范围明确
- [ ] 至少评估了 2 个方案
- [ ] 推荐方案有清晰理由
- [ ] 已识别关键风险
