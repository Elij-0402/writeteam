# 章节创建功能重设计

日期: 2026-03-03

## 问题

当前"添加章节"功能存在设计缺陷：

1. **章节编号 bug**：创建第1、2、3章 → 删除第2章 → 再添加 → 生成"第 3 章"与已有章节重名。根因是编号基于 `filter(chapter).length` 计数，而非最大编号或位置。
2. **仪表板创建是死操作**：创建后不跳转编辑器，用户无法立即写作。
3. **两处创建逻辑不一致**：仪表板用 `chapterCount`（只数 chapter 类型），编辑器用 `documents.length`（所有类型），导致同一项目两处创建出不同编号。
4. **无用户输入**：强制自动命名"第 N 章"，用户无法在创建时自定义标题。
5. **仪表板无重命名入口**：只能进编辑器才能改名。

## 行业调研

| 产品 | 命名方式 | 编号策略 | 删除后处理 |
|------|---------|---------|-----------|
| Scrivener | 用户自由命名 | 编辑时不编号，导出时自动生成 | 无需处理 |
| Sudowrite | 用户自由命名 | 无自动编号 | 无需处理 |
| Dabble | 自动编号 | 实时编号，删除/重排后自动重新编号 | 自动重排 |
| LivingWriter | 用户自由命名 | 无自动编号 | 无需处理 |

结论：主流写作工具以自由命名为主，序号与标题解耦。WriteTeam 当前方式是行业中最差实践。

## 设计方案：自由命名（对标 Scrivener / Sudowrite）

### 核心原则

- **标题是用户的事**：系统不再替用户决定章节叫什么
- **序号是位置的事**：基于 `sort_order` 动态计算，删除/重排自动正确
- **两处入口一致**：创建流程统一，重命名能力统一

### 1. 创建章节交互

**统一创建流程**（仪表板侧边栏 + 编辑器侧边栏共用）：

1. 点击"新建文档" → 弹出对话框
2. 对话框内容：
   - 标题输入框，placeholder="请输入章节标题"，默认值为空
   - 确认/取消按钮
3. 用户不输入标题直接确认 → 使用"未命名章节"作为默认标题
4. 创建成功后：
   - 仪表板侧边栏：自动跳转到编辑器打开新章节
   - 编辑器侧边栏：切换到新章节 + toast 提示
   - 显示 toast "章节已创建"

### 2. 序号显示

- 侧边栏列表中，在标题前动态显示序号：`1. 暴风雨之夜` 或 `2. 未命名章节`
- 序号 = 当前章节在列表中的位置（从1开始），不存入数据库
- 删除、重排后序号自动更新（因为是动态计算的）

### 3. 重命名一致性

- 仪表板 `ProjectTree` 文档操作增加"重命名"菜单项
- 点击后弹出重命名对话框（复用编辑器已有的模式）
- 调用已有的 `renameDocument` server action

### 4. 两处侧边栏操作对比

| 操作 | 仪表板侧边栏 | 编辑器侧边栏 |
|------|-------------|-------------|
| 新建 | 弹对话框 → 创建 → 跳转编辑器 | 弹对话框 → 创建 → 切换文档 |
| 重命名 | hover/菜单 → 重命名对话框 | hover/菜单 → 重命名对话框 |
| 删除 | hover 显示删除图标 | hover 下拉菜单 |
| 排序 | 不支持（导航场景不需要） | 上移/下移 |

### 5. 新建项目默认章节

`createProject` 中创建的默认第一章标题从 `"第 1 章"` 改为 `"未命名章节"`。

## 影响范围

### 需要修改的文件

| 文件 | 变更 |
|------|------|
| `src/components/layout/app-sidebar.tsx` | 新建文档改为弹对话框；创建后导航到编辑器 |
| `src/components/layout/project-tree.tsx` | 文档操作增加"重命名"菜单项；序号动态显示 |
| `src/components/editor/editor-shell.tsx` | 新建文档改为弹对话框；章节编号改为动态序号 |
| `src/app/actions/projects.ts` | 默认章节标题改为"未命名章节" |

### 不需要修改的部分

- 数据库 schema：`title` 字段继续存用户自定义标题
- `sort_order` 逻辑：现有排序逻辑保持不变
- 导出功能：不受影响
- AI 相关代码：不受影响
