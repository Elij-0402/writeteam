# 故事圣经 v3 设计文档 — 编辑器内的活圣经

> 日期: 2026-03-03
> 状态: 已批准
> 范围: 两个独立功能，不过度设计

---

## 1. 背景与动机

故事圣经 v2 已完成：5 标签页结构、4 种 AI 辅助模式、字段级可见性控制、并发冲突检测、世界观子分区、dialogue_style 等。但用户反馈的 4 个核心痛点仍未解决：

1. **入口找不到** — 右侧面板中故事圣经的切换入口不明显，用户不知道怎么进入
2. **AI 不够懂故事** — 写作时无法快速查阅角色设定，需要切换面板打断心流
3. **圣经维护太被动** — 写作中新增的设定需要手动同步回圣经，容易遗忘
4. **缺少一致性保护** — 无实时矛盾检测

本次 v3 只聚焦前两个痛点，采用最小实现。

## 2. 竞品调研核心发现

### 行业验证的功能模式

| 竞品 | 关键功能 | 对 WriteTeam 的启示 |
|------|---------|-------------------|
| NovelCrafter Codex | 编辑器内角色名自动高亮 + 悬浮预览卡 + 别名系统 | **杀手锏功能**，用户不离开编辑器即可查阅圣经条目 |
| Sudowrite | 三栏持久布局，Story Bible 图标常驻 | 圣经入口必须一眼可见 |
| Scrivener | 分屏查阅角色卡，76% 用户改善了角色一致性 | 快速查阅 > 深度编辑 |

### 用户行为关键洞察

- CHI 2025 研究：AI 主动干预在用户**打字时**触发 = 最大反感；在**自然停顿时**触发 = 可接受
- 网文作者最高频的圣经查阅场景：写对话时核查角色口吻（占 40%+）
- 侧边面板常驻 > 独立页面（避免打断写作心流）

## 3. 功能一：圣经入口可见性改造

### 3.1 问题

从用户截图看，右侧面板顶部只有文字标签（如"AI 对话"），故事圣经的切换入口被隐藏在某个下拉或切换中，用户无法快速发现。

### 3.2 方案

右侧面板顶部改为**图标式 Tab 栏**：

```
[ 📖 ] [ 💬 ] [ ✨ ] [ 👁 ]
  圣经   对话   灵感   可视化
```

- 每个面板一个 Lucide 图标，常驻可见
- 当前激活的图标高亮（底部加 accent 色边框）
- 故事圣经图标上显示**小圆点指示**（当有关键字段未填写时，复用 CompletionIndicator 的数据）
- Tooltip 显示面板名称

### 3.3 技术实现

- 修改 `EditorShell`（`src/components/editor/editor-shell.tsx`）中右侧面板的 Tab 切换 UI
- 使用 Lucide 图标：`BookOpen`（圣经）、`MessageSquare`（对话）、`Sparkles`（灵感）、`Eye`（可视化）
- 完成度数据从 `StoryBiblePanel` 的现有 completion 计算逻辑获取

### 3.4 不做的事

- 不新增独立页面或路由
- 不改变 StoryBiblePanel 内部结构
- 不增加新的数据库查询

## 4. 功能二：编辑器内角色悬浮卡

### 4.1 问题

写作时需要查看角色设定（特别是对话风格、性格特征），目前必须切换到故事圣经面板，打断写作心流。

### 4.2 方案

编辑器文本中出现的角色名称 → 自动加下划线标记 → 鼠标悬浮显示角色摘要卡片。

### 4.3 角色摘要卡片内容

从现有 `characters` 表数据提取，不需要新字段：

```
┌──────────────────────────────┐
│  陈安  ·  主角                │
│                              │
│  性格: 沉稳内敛，表面冷漠... │
│  对话风格: 话少但精准...      │
│  外貌: 高瘦，深色短发...      │
│                              │
│  → 查看完整资料               │
└──────────────────────────────┘
```

- 角色名 + role（一行）
- personality（截断 80 字）
- dialogue_style（截断 60 字）
- appearance（截断 60 字）
- "查看完整资料" 链接 → 切换右侧面板到故事圣经角色 Tab 并展开该角色卡

### 4.4 检测逻辑

- 复用 `SaliencyIndicator` 已有的角色名检测结果（已有 5 秒防抖机制）
- 当 SaliencyIndicator 检测到活跃角色列表变化时，更新编辑器中的角色名标记

### 4.5 编辑器标记

- 创建 TipTap `Mark` 扩展 `CharacterMention`
- 该 Mark 给角色名添加 `border-bottom: 1px dashed` 样式（轻量下划线，不干扰阅读）
- Mark 是**装饰性的**，不修改文档内容（不影响保存、不改变 content JSON 结构）
- 用 TipTap 的 `Decoration` Plugin 实现（类似拼写检查的波浪线）

### 4.6 悬浮卡片 UI

- 使用 Radix UI `HoverCard` 组件（项目已有 Radix 依赖）
- 卡片宽度固定 280px，圆角、阴影、暗色模式适配
- 打开延迟 300ms（避免快速划过时频繁弹出）
- 关闭延迟 100ms

### 4.7 数据流

```
SaliencyIndicator 检测活跃角色
  → 角色名列表传递给 TipTap Decoration Plugin
  → Plugin 在编辑器中为匹配文本添加装饰
  → 装饰节点渲染为 HoverCard trigger
  → HoverCard 从 characters 数据（已在 StoryBiblePanel context 中）读取角色信息
```

### 4.8 不做的事

- 不做地点/世界观元素的悬浮卡（第一版只做角色）
- 不做别名系统（留给后续迭代）
- 不做 AI 实时分析（只展示圣经中已有的静态数据）
- 不改变 TipTap 文档内容结构或保存逻辑
- 不做编辑器内的内联编辑（只查看，编辑在圣经面板中）

## 5. 后续迭代方向（不在本次范围）

以下功能基于调研确认了价值，但遵循"不过度设计"原则留给后续：

- **写作反哺圣经** — AI 写作完成后检测新元素，提示添加到圣经
- **圣经健康检查** — 一键 AI 扫描圣经内部矛盾和缺失字段
- **角色别名系统** — 全名/绰号/称呼自动关联
- **结构化角色关系** — relationships 从文本升级为关系标签
- **角色关系可视化图谱** — 复用 @xyflow/react
- **可调节 AI 主动性** — 三档滑块（安静/协作/积极）
- **Progressions 时间线** — 角色/世界观随章节演变
- **网文类型模板** — 升级体系、功法词条等专用字段
