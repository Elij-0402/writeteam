# Phase 3 Agent UX 参考愿景

> 状态：参考文档（非当前实施范围）
> 用途：后期迭代到 Agent 智能体时的 UX 设计参考
> 关联：2026-03-04-writing-agent-design.md

## 核心范式转变

从「工具箱模式」（用户手选 18 个按钮）→「搭档模式」（自然语言对话驱动）

## 关键场景

### 1. 开始新写作日
Agent 主动呈现上文摘要 + 大纲进度，用户无需自行回忆。

### 2. 自然语言指挥创作
用户说"写一个开场，李明去仓库，气氛要压抑" → Agent 自动调用工具收集信息 → 基于角色设定和前文伏笔生成内容。

### 3. 边写边对话
用户自己写一部分 → 随时和 Agent 讨论 → Agent 可以只讨论不写 / 也可以直接写入编辑器 → 用户说"继续"无缝接续。

### 4. 复杂任务分步执行
用户说"帮我写完这章" → Agent 按节拍逐段写 → 每个节拍暂停等反馈 → 报告关键创作决策。

### 5. 随时提问
"张花上次穿什么？" "李明知道她的身份吗？" → Agent 秒回精确答案（基于 RAG + 章节摘要）。

## 界面变化

- AI 面板从 4 个独立 Tab → 1 个统一对话面板
- 工具栏精简为 3 个快捷按钮（续写/编辑/检查）
- 选中文本浮动菜单保留 + 新增自由指令输入

## 两种用户模式（自动识别，无需切换）

- **A 型：我来写，你帮忙** — Agent 被动辅助，只在被叫到时出现
- **B 型：我导演，你来写** — Agent 主动执行，每步暂停等反馈

## 设计原则

- 用户注意力应在故事上，而不是工具选择上
- Agent 能做的事就不要让用户手动做
- 对不支持 tool calling 的模型优雅降级，不拒绝服务
