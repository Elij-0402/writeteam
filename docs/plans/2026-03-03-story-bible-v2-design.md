# 故事圣经 v2 设计文档

> 日期: 2026-03-03
> 状态: 已批准
> 范围: 小步快跑——在现有右侧面板内渐进增强

## 1. 背景与动机

当前故事圣经（StoryBiblePanel）已与全部 17/21 个 AI 功能深度集成，是 WriteTeam 最核心的上下文数据源。但用户反馈四个痛点：

1. **字段太多太散** — 概览 tab 10+ 字段平铺，缺少分组和优先级
2. **缺少 AI 辅助填写** — 所有字段纯手动填写，门槛高
3. **角色管理太简陋** — 扁平文本字段，无关系结构、无对话风格
4. **世界观不够结构化** — 单一文本框，无法分类管理地理/势力/能力等

## 2. 竞品调研结论

| 工具 | 模式 | 借鉴点 |
|------|------|--------|
| Sudowrite | 可折叠卡片 + AI 全流程 | 卡片式角色、AI 填充按钮、Smart Import |
| NovelAI | 关键词触发词条 | 按需激活——只注入相关上下文（WriteTeam 的 Saliency 已实现） |
| World Anvil | Wiki + 28 种模板 | "模板即提示"——字段占位文字即创作引导 |
| Campfire | 18 个模块化面板 | 跨元素链接、关系可视化 |
| Plottr | 标签 + 时间线 | 角色模板库、标签交叉过滤 |

**最佳组合**: Sudowrite 卡片 UX + NovelAI 按需激活 + World Anvil 提示即模板

## 3. 标签页结构（新）

| 标签页 | 内容 | 改动程度 |
|--------|------|---------|
| **概览** | 核心设定 + 故事内容 | 重组分组 |
| **角色** | 可折叠角色卡片 + 关系标签 + dialogue_style | 增强 |
| **世界** | 可折叠子分区卡片（地理/势力/能力/社会/自定义） | 重构 |
| **创作指导** | ai_rules + braindump + outline + notes + prose_mode | 新标签 |
| **AI 可见性** | 保持现有功能 | 不变 |

## 4. 概览标签页

### 4.1 可折叠分组

**「核心设定」**（默认展开）:
- genre — 占位提示: "你的故事属于哪个类型？如：都市悬疑、奇幻冒险、科幻末世..."
- style — 占位提示: "你期望的写作风格？如：紧凑明快、细腻文学、幽默讽刺..."
- pov + tense — 同行排列
- tone — 占位提示: "故事的整体情绪基调？如：压抑、温暖、紧张、荒诞..."

**「故事内容」**（默认展开）:
- synopsis — 占位提示: "用几段话描述你的故事核心情节——AI 会以此为叙事指南"
- themes — 占位提示: "你想探索的核心主题？如：救赎、权力腐化、人性光辉..."

### 4.2 完成度指示器

面板顶部显示环形进度条 + "N/M 已填写"。每个标签页名称后显示该页完成数。

### 4.3 AI 生成按钮

每个文本字段右上角增加 ✨ 按钮，AI 根据已有字段 + 文档内容生成建议。

## 5. 角色标签页

### 5.1 可折叠角色卡片

- 折叠时: `角色名 | 角色定位 | ▸`（一行预览）
- 展开时: 显示所有字段
- 支持拖拽排序

### 5.2 字段升级

保留全部 9 个现有字段，新增:
- **dialogue_style** — "这个角色说话有什么特点？口头禅、语气、用词习惯..."
- **tags** — 自由标签，如"势力A"、"第一卷"、"反派阵营"

### 5.3 关系标签增强

relationships 字段从纯文本改为结构化关系条目:
- 每条关系 = 关系类型 + 目标角色名（如"师徒 — 张三"）
- UI 上用 tag 样式展示

### 5.4 AI 辅助操作

- ✨ AI 生成全部角色（从 synopsis）
- 📄 从文档提取角色（扫描文档文本）
- 每个空字段有 ✨ 填充按钮

### 5.5 占位提示（"模板即提示"）

- personality → "这个角色的核心性格特征是什么？驱动他们行为的内在动力..."
- backstory → "什么过去的经历塑造了这个角色？他们带着什么创伤或信念..."
- goals → "这个角色在故事中想要实现什么？短期目标和终极追求..."
- dialogue_style → "这个角色说话有什么特点？口头禅、语气、措辞偏好..."

## 6. 世界标签页

### 6.1 可折叠子分区卡片

| 子分区 | 占位提示 |
|--------|---------|
| 地理环境 | "故事发生在什么样的地方？气候、地形、重要地标..." |

## 8. AI 辅助系统

### 8.1 新端点

```
POST /api/ai/bible-assist
{
  projectId: string
  mode: "field-generate" | "braindump-expand" | "document-extract" | "character-generate"
  targetField?: string
  currentBible: Partial<StoryBibleData>
  documentTexts?: string[]
}
```

### 8.2 四种模式

1. **field-generate** — 为单个字段生成建议（参考已有字段）
2. **braindump-expand** — 将灵感池拆解为各字段建议（checklist 返回）
3. **document-extract** — 从文档文本提取角色 + 设定 + 世界观
4. **character-generate** — 从 synopsis 批量生成角色卡片

### 8.3 预览确认

所有 AI 生成结果以预览形式展示，用户需明确「采纳」才写入。
